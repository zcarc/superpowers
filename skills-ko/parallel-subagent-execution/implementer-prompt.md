# 병렬 구현자 서브에이전트 프롬프트 템플릿 (Parallel Implementer Subagent Prompt Template)

병렬 웨이브 내의 작업을 위해 구현자 서브에이전트를 파견할 때 이 템플릿을 사용하십시오.

```
Task 도구 (범용):
  description: "작업 N 구현: [작업 이름]"
  prompt: |
    귀하는 병렬 실행 웨이브의 일부로 작업 N: [작업 이름]을 구현하고 있습니다.

    ## 작업 설명 (Task Description)

    [계획에서 추출한 작업의 전체 텍스트 - 여기 붙여넣으십시오, 서브에이전트가 직접 파일을 읽게 하지 마십시오]

    ## 문맥 (Context)

    [상황 설정: 이 작업이 어디에 적합한지, 이미 충족된 의존성, 아키텍처 문맥]

    ## 파일 소유권 (File Ownership)

    파일 소유권은 운영 체제나 사용자의 권한이 아니라, 이 작업을 위해 할당된 파일 범위를 의미합니다.

    귀하는 이 작업을 위해 다음 파일들만 소유합니다:
    - [exact/path/one]
    - [exact/path/two]

    다른 서브에이전트들이 다른 작업에서 병렬로 작업 중일 수 있습니다.

    할당된 범위 밖의 파일을 수정하지 마십시오. 할당된 범위 밖의 변경 사항을 원복하지 마십시오.
    차단(BLOCKED)되었거나 문맥이 필요(NEEDS_CONTEXT)한 경우 멈추고 보고하십시오.

    ## 할당된 브랜치 및 작업트리 (Assigned Branch and Worktree)

    할당된 브랜치와 작업트리에서만 작업하십시오:
    - 브랜치: [브랜치 이름]
    - 작업트리: [작업트리 경로]

    다른 작업의 브랜치나 작업트리로 전환하지 마십시오.
    다른 작업 브랜치를 직접 통합하지 마십시오.

    ## 인터페이스 계약 (Interface Contract)

    [합의된 공유 인터페이스, 타입, 라우트, 컴포넌트 API, 스키마 또는 계약]

    계약이 잘못되었거나 불완전해 보이면, 진행하기 전에 멈추고 질문하십시오.

    ## 시작하기 전 확인 사항 (Before You Begin)

    요구 사항, 소유권, 테스트 또는 인터페이스 계약에 대해 불분명한 사항이 있으면 지금 질문하십시오. 추측하지 마십시오.

    ## 귀하의 역할 (Your Job)

    요구 사항이 명확해지면:
    1. 작업에 명시된 내용을 정확하게 구현하십시오.
    2. 테스트를 작성하십시오 (작업에서 지시한 경우 TDD를 따르십시오).
    3. 구현이 작동하는지 확인하십시오.
    4. 작업을 커밋하십시오.
    5. 자체 검토(Self-review)를 수행하십시오.
    6. 결과를 보고하십시오.

    작업 디렉토리: [directory]

    ## 자체 검토 (Self-Review)

    이것은 로컬 구현자 확인 사항이며 워크플로우의 공식 리뷰가 아닙니다.

    보고하기 전에 다음을 확인하십시오:
    - 할당된 파일 범위 내에 머물렀는가?
    - 할당된 브랜치와 작업트리 내에 머물렀는가?
    - 작업을 완전히 구현했는가?
    - 테스트가 실제 동작을 검증하는가?
    - 다른 작업의 미완성 작업에 대한 가정을 도입하지 않았는가?

    보고하기 전에 수정할 수 있는 모든 것을 수정하십시오.

    ## 보고 형식 (Report Format)

    - **상태:** DONE | DONE_WITH_CONCERNS | BLOCKED | NEEDS_CONTEXT
    - 구현한 내용
    - 테스트 내용 및 결과
    - 변경된 파일
    - 소유권 또는 통합에 관한 우려 사항
    - 자체 검토 결과
```
