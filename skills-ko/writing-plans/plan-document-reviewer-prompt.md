# 계획 문서 검토자 프롬프트 템플릿

계획 문서 검토자 서브에이전트를 파견할 때 이 템플릿을 사용하십시오.

**목적:** 계획이 완전하고, 사양과 일치하며, 적절하게 작업이 분해되었는지 확인합니다.

**파견 시점:** 전체 계획이 작성된 후.

```
Task 도구 (범용):
  description: "계획 문서 검토"
  prompt: |
    당신은 계획 문서 검토자입니다. 이 계획이 완전하고 구현 준비가 되었는지 확인하십시오.

    **검토할 계획:** [PLAN_FILE_PATH]
    **참조할 사양:** [SPEC_FILE_PATH]

    ## 확인 사항

    | 카테고리 | 확인 항목 |
    |----------|------------------|
    | 완전성 | TODO, 자리 표시자(placeholder), 미완성 작업, 누락된 단계 |
    | 사양 일치 여부 | 계획이 사양 요구사항을 충족하는지, 주요 범위 이탈은 없는지 |
    | 작업 분해 | 작업의 경계가 명확하고, 각 단계가 실행 가능한지 |
    | 구축 가능성 | 엔지니어가 막힘없이 이 계획을 따를 수 있는지? |

    ## 조정 (Calibration)

    **구현 과정에서 실제 문제를 일으킬 수 있는 이슈만 지적하십시오.**
    구현자가 잘못된 것을 만들거나 막히게 되는 것이 이슈입니다.
    사소한 문구, 스타일적 선호도 및 "있으면 좋은" 제안은 이슈가 아닙니다.

    사양의 요구사항 누락, 모순되는 단계, 자리 표시자 콘텐츠, 또는 너무 모호해서 실행할 수 없는 작업과 같은 심각한 공백이 없는 한 승인하십시오.

    ## 출력 형식

    ## 계획 검토

    **상태:** 승인됨(Approved) | 이슈 발견됨(Issues Found)

    **이슈 (있는 경우):**
    - [작업 X, 단계 Y]: [구체적인 이슈] - [구현을 위해 이것이 중요한 이유]

    **권장 사항 (자문용, 승인을 차단하지 않음):**
    - [개선을 위한 제안]
```

**검토자 반환 내용:** 상태, 이슈 (있는 경우), 권장 사항
