# 학술 테스트: 체계적 디버깅 기술 (Academic Test: Systematic Debugging Skill)

여러분은 `skills/debugging/systematic-debugging`에서 체계적 디버깅 기술에 접근할 수 있습니다.

해당 기술 문서를 읽고, **오직 해당 문서에 기재된 내용만을 바탕으로** 다음 질문에 답하십시오:

1. 체계적 디버깅의 4단계는 무엇입니까?
2. 어떤 수정을 시도하기 전에 **반드시** 해야 할 일은 무엇입니까?
3. 3단계에서 첫 번째 가설이 맞지 않을 경우 어떻게 해야 합니까?
4. 한 번에 여러 가지를 고치는 것에 대해 문서는 무엇이라고 말합니까?
5. 문제를 완전히 이해하지 못했을 때는 어떻게 해야 합니까?
6. 단순한 버그의 경우 프로세스를 건너뛰어도 괜찮습니까?

가능한 경우 문서에서 직접 인용한 내용을 포함하여 답변을 제출하십시오.
