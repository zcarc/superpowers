# 코드 품질 리뷰어 프롬프트 템플릿 (Code Quality Reviewer Prompt Template)

코드 품질 리뷰어 서브에이전트를 파견할 때 이 템플릿을 사용하십시오.

**목적:** 구현이 잘 되었는지(깔끔함, 테스트 완료, 유지보수 가능) 확인합니다.

**사양 준수 리뷰가 통과된 후에만 파견하십시오.**

```
Task 도구 (superpowers:code-reviewer):
  requesting-code-review/code-reviewer.md의 템플릿을 사용하십시오.

  WHAT_WAS_IMPLEMENTED: [구현자의 보고서 내용]
  PLAN_OR_REQUIREMENTS: [계획 파일]의 작업 N
  BASE_SHA: [작업 전 커밋]
  HEAD_SHA: [현재 커밋]
  DESCRIPTION: [작업 요약]
```

**표준 코드 품질 사항 외에도 리뷰어는 다음을 확인해야 합니다:**
- 각 파일이 명확한 인터페이스와 함께 하나의 책임을 수행하는가?
- 독립적으로 이해되고 테스트될 수 있도록 기능이 분해(Decomposed)되었는가?
- 구현이 계획의 파일 구조를 따르고 있는가?
- 이번 구현으로 생성된 새 파일이 이미 너무 크거나, 기존 파일이 크게 확장되었는가? (기존 파일 크기가 아닌, 이번 변경 사항이 기여한 부분에 집중하십시오.)

**코드 리뷰어 반환 형식:** 장점, 이슈 (치명적/중요/사소함), 평가
