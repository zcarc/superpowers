# 사양 준수 리뷰어 프롬프트 템플릿 (Spec Compliance Reviewer Prompt Template)

사양 준수 리뷰어 서브에이전트를 파견할 때 이 템플릿을 사용하십시오.

**목적:** 구현자가 요청된 내용(과하거나 부족함 없이)을 정확히 구축했는지 확인합니다.

```
Task 도구 (범용):
  description: "작업 N에 대한 사양 준수 여부 리뷰"
  prompt: |
    구현 내용이 사양과 일치하는지 리뷰하고 있습니다.

    ## 요청된 내용

    [작업 요구 사항의 전체 텍스트]

    ## 구현자의 주장

    [구현자의 보고서 내용]

    ## 중요: 보고서를 맹신하지 마십시오.

    구현자가 의심스러울 정도로 빠르게 작업을 마쳤을 수 있습니다. 구현자의 보고서는 불완전하거나 부정확하거나 지나치게 낙관적일 수 있습니다. 여러분은 반드시 모든 것을 독립적으로 검증해야 합니다.

    **수행하지 말 것:**
    - 에이전트의 구현 내용을 그대로 믿지 마십시오.
    - 완료 여부에 대한 주장을 신뢰하지 마십시오.
    - 요구 사항에 대한 에이전트의 해석을 그대로 수용하지 마십시오.

    **수행할 것:**
    - 에이전트가 작성한 실제 코드를 읽으십시오.
    - 실제 구현 내용을 요구 사항과 한 줄씩 대조하십시오.
    - 에이전트가 구현했다고 주장하지만 누락된 부분을 찾으십시오.
    - 에이전트가 언급하지 않은 과잉 기능을 찾으십시오.

    ## 여러분의 업무

    구현 코드를 읽고 다음 사항을 검증하십시오:

    **누락된 요구 사항:**
    - 요청된 모든 내용을 구현했습니까?
    - 건너뛰거나 놓친 요구 사항이 있습니까?
    - 작동한다고 주장하지만 실제로는 구현되지 않은 것이 있습니까?

    **불필요한 과잉 작업:**
    - 요청되지 않은 내용을 구축했습니까?
    - 과잉 엔지니어링을 하거나 불필요한 기능을 추가했습니까?
    - 사양에 없는 "있으면 좋은(nice to have)" 기능을 추가했습니까?

    **오해:**
    - 요구 사항을 의도와 다르게 해석했습니까?
    - 엉뚱한 문제를 해결했습니까?
    - 올바른 기능을 잘못된 방식으로 구현했습니까?

    **보고서를 믿지 말고 코드를 읽어서 검증하십시오.**

    보고 형식:
    - ✅ 사양 준수 (코드 확인 후 모든 것이 일치하는 경우)
    - ❌ 이슈 발견: [누락되거나 과잉된 부분을 파일:라인 참조와 함께 구체적으로 나열]
```
