# 플랜 실행 (Executing Plans)

## 개요

작성된 구현 계획을 메인 에이전트에서 직접, 단계별로 실행하며, 일반적인 검증을 수행하고 필요한 경우에만 명시적으로 핸드오프합니다.

**시작 시 선언:** "이 계획을 구현하기 위해 executing-plans 기술을 사용하고 있습니다."

**경계:** 이 기술은 '인라인 실행(Inline Execution)'이 선택되었을 때만 적용됩니다. 선택된 모드가 '하위 에이전트 기반(Subagent-Driven)' 또는 '병렬 하위 에이전트(Parallel Subagents)'인 경우, 해당 기술을 대신 사용하십시오.

## 프로세스

### 1단계: 계획 로드 및 검토
1. 계획 파일 읽기
2. 비판적으로 검토 - 계획에 대한 질문이나 우려 사항 식별
3. 우려 사항이 있는 경우: 시작하기 전에 파트너에게 제기
4. 우려 사항이 없는 경우: TodoWrite를 생성하고 진행

### 2단계: 작업 실행

각 작업에 대해:
1. in_progress로 표시
2. 각 단계를 정확히 수행 (계획에는 한 입 크기의 단계들이 있음)
3. 명시된 대로 검증 실행
4. 필요한 로컬 자체 검토를 수행하되, 이 워크플로우에서는 일상적인 작업 단위 공식 리뷰를 파견하지 마십시오.
5. 완료로 표시

### 3단계: 최종 공식 리뷰 게이트

모든 작업이 완료되고 검증된 후:
- 이 워크플로우에서는 일상적인 작업 단위 공식 리뷰를 파견하지 마십시오.
- 다음을 알립니다: "완료된 구현을 검토하기 위해 requesting-code-review 기술을 사용하고 있습니다."
- 그런 다음 `superpowers:requesting-code-review`를 사용하여 완료된 전체 통합 결과물을 검토하십시오.
- 리뷰 결과(findings)가 반환되면, `superpowers:receiving-code-review`를 사용하여 이를 평가하고 '중요(Important)' 또는 '심각(Critical)' 문제를 진행하기 전에 해결하십시오.
- 실행 가능한(actionable) 리뷰 결과가 없다면 계속 진행하십시오.

### 4단계: 선택적 통합 핸드오프

사용자가 커밋, 머지, 푸시 또는 PR 생성과 같은 통합 작업을 명시적으로 요청하지 않는 한 `superpowers:finishing-a-development-branch`를 호출하지 마십시오.

사용자가 명시적으로 통합 작업을 요청하는 경우:
- 다음을 알립니다: "이 작업을 완료하기 위해 finishing-a-development-branch 기술을 사용하고 있습니다."
- 그런 다음 `superpowers:finishing-a-development-branch`를 사용합니다.

## 중단 및 도움 요청 시점

**다음과 같은 경우 즉시 실행을 중단하십시오:**
- 차단 요소 발생 (누락된 종속성, 테스트 실패, 불명확한 지침)
- 시작을 방해하는 치명적인 계획 공백 발견
- 지침을 이해할 수 없음
- 검증이 반복적으로 실패함

**추측하기보다 명확한 설명을 요청하십시오.**

## 이전 단계 재방문 시점

**다음과 같은 경우 검토(1단계)로 돌아가십시오:**
- 피드백에 따라 파트너가 계획을 업데이트함
- 근본적인 접근 방식의 재고가 필요함

**차단 요소를 억지로 통과하려 하지 마십시오** - 중단하고 질문하십시오.

## 기억할 것
- 먼저 계획을 비판적으로 검토하십시오.
- 계획의 단계를 정확히 따르십시오.
- 검증을 건너뛰지 마십시오.
- 계획에서 언급할 때 해당 기술을 참조하십시오.
- 막히면 중단하고 추측하지 마십시오.
- 사용자의 명시적인 동의 없이 main/master 브랜치에서 구현을 시작하지 마십시오.

## 통합

**필요한 워크플로우 기술:**
- **superpowers:using-git-worktrees** - 필수: 시작 전 격리된 작업 공간 설정
- **superpowers:writing-plans** - 이 기술이 실행할 계획을 생성
- **superpowers:requesting-code-review** - 필수: 완료된 전체 통합 결과물에 대한 최종 공식 리뷰.
- **superpowers:receiving-code-review** - 최종 공식 리뷰에서 검토가 필요한 피드백을 반환할 때 사용합니다.
- **superpowers:finishing-a-development-branch** - 명시적인 통합 작업 시에만 사용
