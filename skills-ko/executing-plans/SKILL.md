# 계획 실행 (Executing Plans)

## 개요

계획을 로드하고, 비판적으로 검토하며, 모든 작업을 실행하고, 완료 시 실제 검증된 상태를 보고합니다.

**시작 시 공지:** "이 계획을 구현하기 위해 executing-plans 기술을 사용하고 있습니다."

**참고:** 인라인 실행(Inline execution)은 메인 에이전트가 현재 세션에서 작업을 직접 구현하고 검증하는 것을 의미합니다. 인라인 경로의 일부로 서브에이전트가 포함되어 있다고 가정하지 마십시오.

## 프로세스

### 1단계: 계획 로드 및 검토
1. 계획 파일 읽기
2. 비판적으로 검토 - 계획에 대한 질문이나 우려 사항 식별
3. 우려 사항이 있는 경우: 시작하기 전에 파트너(사용자)에게 제기
4. 우려 사항이 없는 경우: Todo 작성 후 진행

### 2단계: 작업 실행

각 작업에 대해:
1. `in_progress`로 표시
2. 각 단계를 정확히 따름 (계획에는 한입 크기의 단계들이 있음)
3. 명시된 대로 검증 실행
4. 완료로 표시

### 3단계: 선택적 리뷰 게이트 (Optional Review Gate)

모든 작업이 완료되고 검증된 후:
- 인라인 실행의 경우, `requesting-code-review`를 자동으로 호출하지 마십시오.
- 다음의 경우에만 리뷰를 요청하십시오:
  - 사용자가 명시적으로 리뷰를 요청할 때
  - 변경 사항이 `superpowers:requesting-code-review`에 정의된 리뷰 임계값을 넘을 때
  - 영속성(persistence), 데이터 모델 또는 다중 서피스 통합 리스크가 포함된 경우
- 리뷰가 요청되거나 리스크로 인해 정당화되는 경우, 공지하십시오: "완료된 구현을 검토하기 위해 requesting-code-review 기술을 사용하고 있습니다."
- 그 다음 `superpowers:requesting-code-review`를 사용하고, 계속 진행하기 전에 중요(Important) 또는 치명적(Critical) 문제를 해결하십시오.

### 4단계: 통합 작업 (Integration Actions)

사용자가 다음 작업을 명시적으로 요청하지 않는 한 `finishing-a-development-branch`를 호출하지 마십시오:
- 커밋(commit)
- 머지(merge)
- 푸시(push)
- PR 생성

사용자가 이러한 통합 작업 중 하나를 명시적으로 요청하는 경우:
- 공지하십시오: "이 작업을 완료하기 위해 finishing-a-development-branch 기술을 사용하고 있습니다."
- 그 다음 `superpowers:finishing-a-development-branch`를 사용하십시오.

## 중단 및 도움 요청 시기

**다음과 같은 경우 즉시 실행을 중단하십시오:**
- 차단 요소 발생 (누락된 의존성, 테스트 실패, 지침 불분명)
- 시작을 방해하는 계획의 중대한 결함 발견
- 지침을 이해하지 못함
- 검증이 반복적으로 실패함

**추측하기보다 명확한 설명을 요청하십시오.**

## 이전 단계로 돌아가야 할 때

**다음과 같은 경우 검토(1단계)로 돌아가십시오:**
- 파트너가 여러분의 피드백을 바탕으로 계획을 업데이트함
- 근본적인 접근 방식의 재고가 필요함

**차단 요소를 억지로 통과하려 하지 마십시오** - 중단하고 질문하십시오.

## 기억할 것
- 먼저 계획을 비판적으로 검토하십시오.
- 계획의 단계를 정확히 따르십시오.
- 검증을 건너뛰지 마십시오.
- 계획에서 언급할 때 해당 기술을 참조하십시오.
- 막히면 중단하고 추측하지 마십시오.
- 사용자의 명시적인 동의 없이 main/master 브랜치에서 구현을 시작하지 마십시오.

## 통합

**필요한 워크플로우 기술:**
- **superpowers:using-git-worktrees** - 필수: 시작 전 격리된 작업 공간 설정
- **superpowers:writing-plans** - 이 기술이 실행할 계획을 생성
- **superpowers:requesting-code-review** - 인라인 실행에서는 선택 사항이며, 리뷰가 요청되거나 리스크가 있을 때 사용
- **superpowers:finishing-a-development-branch** - 명시적인 통합 작업 시에만 사용
